# 任务完成检查清单

> 每个任务完成前的质量检查

## 代码质量

- [ ] 所有函数都有中文 docstring
- [ ] 复杂逻辑有适当注释
- [ ] 类型提示完整
- [ ] 遵循命名规范
- [ ] 没有显而易见的注释

## 架构检查

- [ ] 代码放在正确的 DDD 层级
- [ ] 依赖抽象而非具体实现
- [ ] 单一职责原则
- [ ] 适当的错误处理和日志

## 异步编程

- [ ] 使用 async/await 进行 I/O
- [ ] HTTP 请求有超时控制
- [ ] 网络请求有重试机制
- [ ] 异常被正确捕获

## 数据库操作

- [ ] 使用 SQLAlchemy 异步 API
- [ ] 批量操作优化性能
- [ ] 适当的索引使用

## 测试

- [ ] 测试文件放在正确位置
- [ ] 测试覆盖核心逻辑
- [ ] 测试命名清晰

## 文档

- [ ] 复杂功能有设计文档
- [ ] README 更新（如必要）
- [ ] 环境变量示例更新

## Git 提交前

- [ ] 代码已测试通过
- [ ] 没有调试代码
- [ ] 没有敏感信息
- [ ] commit message 清晰

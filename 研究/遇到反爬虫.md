#### 测试 2: Playwright 验证

**测试方法：**
使用真实浏览器（Playwright）访问网站。

**结果：**
- ✅ 状态码: `200 OK`
- ✅ 页面加载成功
- ✅ 内容包含新闻关键词
- 🔑 关键 Cookie: `cf_clearance=xxxxx...`

**过程分析：**
1. Playwright 访问 → Cloudflare 返回 JavaScript 挑战
2. 浏览器自动执行 JavaScript
3. Cloudflare 设置 `cf_clearance` cookie
4. 重定向到真实页面

**结论：** 网站要求 **JavaScript 执行**才能通过验证。

---

#### 测试 3: 复用 Playwright Cookie

**测试方法：**
将 Playwright 获取的完整请求头（包括 `cf_clearance` cookie）用于 curl_cffi。

**Cookie 示例：**
```
cf_clearance=LlAtQKA92c7lbckLJHTGnt348OohIfESXtYbWB7OnKY-1773287543-1.2.1.1-...
```

**结果：**
| 测试方案 | 状态码 | 结果 | 说明 |
|---------|--------|------|------|
| Playwright Headers + Cookie | 403 | ❌ 失败 | Cookie 会话绑定 |
| 只用 Playwright Headers | 403 | ❌ 失败 | 需要 cookie |
| Safari 指纹 + Cookie | 403 | ❌ 失败 | 指纹不匹配 |

**失败原因：**
1. **Cookie 会话绑定**
   - Cloudflare 验证 TLS 指纹、浏览器指纹、IP 地址
   - Cookie 无法跨会话/客户端复用
2. **浏览器指纹不匹配**
   - curl_cffi 的 TLS 指纹与 Playwright 不同
   - Cloudflare 检测到不一致后拒绝请求

**结论：** 无法通过 Cookie 复用绕过。

### 技术分析

#### Cloudflare Bot Management 工作原理

```
用户请求 → Cloudflare 拦截
    ↓
返回 JavaScript 挑战
    ↓
浏览器执行 JS → 生成验证信息
    ↓
Cloudflare 验证 → 设置 cf_clearance cookie
    ↓
允许访问真实页面
```

**关键点：**
- JavaScript 挑战无法由非浏览器客户端执行
- Cookie 与浏览器会话深度绑定（TLS 指纹、Canvas 指纹等）
- curl_cffi 只能模拟 TLS 指纹，无法执行 JavaScript

#### 为什么 Playwright 成功但 curl_cffi 失败？

| 特性 | Playwright | curl_cffi |
|-----|-----------|----------|
| TLS 指纹伪装 | ✅ 真实浏览器 | ✅ 模拟浏览器 |
| JavaScript 执行 | ✅ 支持 | ❌ 不支持 |
| Canvas/WebGL 指纹 | ✅ 真实 | ❌ 无 |
| Cookie 生成 | ✅ JS 挑战后生成 | ❌ 无法获取 |

**结论：** 该站点的 Cloudflare 配置要求 JavaScript 验证，这是 curl_cffi 的根本限制。

### 可行性评估

#### ❌ 不可行的方案

| 方案 | 原因 |
|-----|------|
| curl_cffi 优化配置 | 无法执行 JavaScript |
| 优化请求头 | 验证机制在 JS 层面 |
| 代理 IP 轮换 | 仍然无法通过 JS 挑战 |
| Cookie 复用 | 会话绑定，无法跨客户端 |

#### ✅ 可行的方案

**方案 1: 实现 PlaywrightAdapter（推荐）**

创建新的 HTTP 适配器，使用真实浏览器：

```python
# src/v1/DDD/infrastructure/http/playwright_adapter.py

class PlaywrightAdapter(BaseHttpAdapter):
    """使用 Playwright 处理需要 JavaScript 的站点"""

    async def send(self, request_config: RequestParameter) -> Response:
        page = await self.browser.new_page()
        response = await page.goto(request_config.url)
        content = await page.content()
        await page.close()
        return Response(
            status_code=response.status,
            text=content,
            headers=dict(response.headers),
            url=response.url,
        )
```

**优势：**
- ✅ 完全模拟真实浏览器行为
- ✅ 自动处理 JavaScript 挑战
- ✅ 支持复杂的动态内容

**劣势：**
- ❌ 资源消耗大（每次请求启动浏览器）
- ❌ 速度慢（比 HTTP 请求慢 5-10 倍）
- ❌ 需要额外依赖（Playwright + Chromium）

**配置示例：**
```python
# bn_borneo_bulletin_config.py
http_adapter_overrides={
    "adapter_type": "playwright",
    "browser": "chromium",
    "headless": True,
    "timeout": 60.0,
}
```

## 通用反爬虫策略

### 防护类型分级

| 防护类型 | 难度 | 绕过方法 | 成功率 |
|---------|------|---------|--------|
| 无防护 | 🟢 简单 | httpx 标准请求 | 100% |
| User-Agent 检查 | 🟢 简单 | 添加 UA header | 100% |
| 频率限制 | 🟡 中等 | 请求延迟 + 随机化 | 95% |
| 标准 Cloudflare | 🟡 中等 | curl_cffi 即可 | 85% |
| Cloudflare Bot Management | 🔴 困难 | curl_cffi + 优化 | 40% |
| Bot Management + JS 挑战 | 🔴 非常困难 | Playwright | 90% |

### 何时需要 Playwright？

**必须使用 Playwright 的场景：**
1. ✅ 网站返回 JavaScript 挑战页面
2. ✅ 内容完全由 JavaScript 渲染（SPA）
3. ✅ 需要模拟用户交互（点击、滚动）
4. ✅ 需要通过 reCAPTCHA 等验证

**不建议使用 Playwright 的场景：**
1. ❌ 静态 HTML 内容（curl_cffi 即可）
2. ❌ 有 RSS Feed 可用
3. ❌ 高频爬取（Playwright 太慢）

### 附录：测试脚本模板

#### 快速测试 curl_cffi

```python
import asyncio
from curl_cffi.requests import AsyncSession

async def test_site(url: str):
    impersonates = ["safari15_5", "chrome120", "edge101"]

    for imp in impersonates:
        print(f"\n测试 {imp}...")
        session = AsyncSession(impersonate=imp)

        try:
            resp = await session.get(
                url,
                headers={
                    "Accept": "text/html,application/xhtml+xml,application/xml;q=0.9,*/*;q=0.8",
                    "Referer": "https://www.google.com/",
                },
                timeout=30,
            )

            status = resp.status_code
            is_cloudflare = any([
                "cloudflare" in resp.text.lower(),
                "just a moment" in resp.text.lower(),
            ])

            if status == 200 and not is_cloudflare:
                print(f"[SUCCESS] {imp} - 状态 {status}")
                return True
            else:
                print(f"[FAIL] {imp} - 状态 {status} | Cloudflare: {is_cloudflare}")

        except Exception as e:
            print(f"[ERROR] {imp} - {e}")

        finally:
            await session.close()

    return False

# 使用
asyncio.run(test_site("https://example.com"))
```
